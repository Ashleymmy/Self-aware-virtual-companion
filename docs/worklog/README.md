# 工作日志说明

用于记录我们每次协作对话的目标、执行内容、修改文件和验证结果。

## 目录约定

- 日志索引：`docs/worklog/INDEX.md`
- 每日日志：`docs/worklog/YYYY-MM-DD.md`

## 记录模板

每次新增一条记录，建议包含以下字段：

- `对话目标`：本次你提出的需求
- `执行内容`：我具体做了什么
- `修改文件`：新增/修改/删除的文件路径
- `验证结果`：执行过的命令与结果
- `备注`：待办或风险说明（可选）

## 维护规则

- 每次对话结束后追加一条记录
- 文件路径使用仓库相对路径
- 内容以事实为主，避免冗长
