  - 处理配置盘点中发现的生产环境存储路径漂移，避免 `savc-ui` 写入路径与 `docker-compose.prod.yml` 挂载路径不一致。

- 执行内容
  - 复核 `packages/ui/vite.config.ts` 与 `packages/ui/storage/global-storage.ts`，确认 `SavcGlobalStorageService` 以 `packages/` 为根，实际写入目录是 `packages/config/storage`。
  - 复核 `packages/config/storage/savc-disaster-backup.yaml`，确认运行日志中的 SQLite/YAML 实际路径也是 `/workspace/packages/config/storage/...`。
  - 将 `infra/docker/docker-compose.prod.yml` 中 `savc-ui` 与 `savc-proactive` 的 `savc_storage_data` 卷挂载点，从 `/workspace/config/storage` 改为 `/workspace/packages/config/storage`，与实际运行路径保持一致。

- 修改文件
  - `infra/docker/docker-compose.prod.yml`
  - `docs/worklog/2026-03-14.md`

- 验证结果
  - `rg -n "/workspace/config/storage|packages/config/storage|savc_storage_data" infra packages`：通过，确认生产 compose 原先挂载到 `/workspace/config/storage`，而运行态落盘路径为 `/workspace/packages/config/storage`。
  - `docker compose -f infra/docker/docker-compose.prod.yml config`：通过，修正后 compose 配置可正常展开。

- 备注
  - 这是生产部署层的路径修正，不影响当前已运行的 cloud/dev compose。
  - 若此前已经用旧版生产 compose 跑过容器，旧卷里可能没有被实际消费的数据；真正被 `savc-ui` 使用的是修正后的 `packages/config/storage` 路径。
