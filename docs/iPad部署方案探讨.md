# iPad 部署 OpenClaw 方案探讨

> 创建日期: 2026-02-05
> 状态: **搁置中** - 待后续讨论
> 项目: Self-aware Virtual Companion (SAVC)

---

## 背景

希望将 OpenClaw 虚拟人部署到 iPad 上，实现随身携带、随时交互的体验。

---

## 核心挑战

OpenClaw 基于 **Node.js/TypeScript**，而 iOS/iPadOS 有严格限制：

| 限制 | 说明 |
|------|------|
| 无法运行 Node.js | iOS 沙箱不允许 |
| 无后台守护进程 | 应用切后台会被挂起 |
| 文件系统受限 | 只能访问 App 沙箱内文件 |

---

## 可行方案

### 方案 1: iPad + 云服务器
iPad 作为客户端，OpenClaw 运行在云端。
- 优点：简单、稳定
- 缺点：隐私风险、需付费

### 方案 2: iPad + 家庭服务器 ⭐推荐
树莓派/Mac Mini/NAS 运行 OpenClaw，通过 Tailscale 连接。
- 优点：隐私安全、可随时连接
- 缺点：需维护服务器

### 方案 3: iSH/a-Shell
iOS 上的 Linux 模拟器。
- 结论：性能太差，不可行

### 方案 4: WebContainer
浏览器中运行 Node.js。
- 结论：功能受限，仅适合演示

### 方案 5: 原生 iOS App (长期)
用 Swift 重写核心逻辑。
- 优点：最佳体验
- 缺点：开发量大

### 方案 6: 越狱
- 结论：不推荐，风险大
